# Part 1: 現状分析

## 1. 実機確認サマリー

v3.0 完了後、390px（iPhone 14）/ 375px（iPhone SE）で全対象ページのスクリーンショットを撮影し、問題点を洗い出した。

> スクリーンショット: `docs/qa/v3.1/` に保存

### 1-1. ダッシュボード（390px）

**良い点**:
- ウェルカムカードのグラデーションは見映えが良い
- 学習コース・スキルアップのカードは 1 カラムで表示され、致命的な崩れはない
- 学習ステータスセクションのスタッツカードは 2 列で収まっている
- ヒートマップが画面幅に収まっている

**問題点**:
- ウェルカムカード内の余白が大きすぎる（Coden ロゴが場所を取りすぎ）
- 「学習コース」セクションのカテゴリカード（React, TypeScript）がフラットすぎて、モバイルでの視認性が低い
- 「ベースヌック」「デイリーチャレンジ」カードはカテゴリカードと見た目が異なり（背景色付き）、情報の階層が不統一
- 「スキルアップ」セクションの 4 カード（コードドクター等）も 1 カラムだが、アイコン+タイトル+説明 の縦積みが冗長
- 学習ステータスの「次の目標まで」プログレスバーが小さすぎて見づらい
- 全体的にスクロール量が多い（ページ全体が非常に長い）

### 1-2. カリキュラム（390px）

**良い点**:
- アコーディオン開閉は正常に動作
- ステップ一覧は 1 カラムで見やすい
- ロック状態の表示（モバイルでは絵文字のみ）がコンパクト

**問題点**:
- カテゴリヘッダー（React / TypeScript）とコースアコーディオンの視覚的な階層が弱い
- 練習モードセクションのカード（デイリー、コードドクター等）がダッシュボードと重複した内容
- ステップリンクのタップターゲットが行全体ではなくテキスト部分のみに見える
- アコーディオンの `px-5 py-4` パディングがモバイルではやや大きい

### 1-3. プロフィール（390px）

**良い点**:
- 表示名入力 + 保存ボタンは問題なく動作
- スタッツカード（総ポイント・連続学習等）は 1 カラムで表示
- 実績バッジは 1 カラムでリスト形式、見やすい

**問題点**:
- スタッツカード 4 枚が全て 1 カラムで縦に並び、スクロール量が多い（2×2 グリッドが妥当）
- 実績バッジ 23 個がフラットに一覧されており、スクロールが非常に長い
- 未獲得バッジにグレーアウト処理があるが、獲得済みとの視覚的差異が小さい
- ポイント履歴が下端に埋もれている
- 「ダッシュボードへ戻る」リンクの配置が右上で、モバイルでは気づきにくい

### 1-4. ログイン / サインアップ（390px）

**良い点**:
- フォームは画面中央に配置され、基本的なレイアウトは問題ない
- 「アカウントを作成」「ログインはこちら」のリンクは視認可能

**問題点**:
- ページ上部の余白（`py-16`）が大きすぎ、フォームが画面下半分に追いやられている
- 入力フィールドの高さ（`py-2` ≒ 32px）が 44px タップターゲットに未達
- ボタンとフィールドの間隔が均等すぎて、視覚的なグループ化が弱い
- エラーメッセージの表示位置・スタイルがモバイルで最適化されていない
- Coden ロゴ + タイトルの配置がやや単調

### 1-5. AppHeader ドロワー（390px）

**良い点**:
- ドロワーの開閉は正常に動作
- ナビゲーションリンクは網羅的（全ページへのリンクあり）
- ユーザー名 + ポイント + 連続学習の表示がコンパクト
- ログアウトボタンが下端に固定

**問題点**:
- ドロワー幅が `w-72`（288px）固定で、画面左側に背景のページが見えてしまい中途半端
- ナビゲーションリンクのタップターゲットがテキスト幅のみ（行全体ではない）
- カテゴリ名（React, TypeScript）とサブリンク（デイリー、コードドクター等）の階層表現が弱い
- 「メニュー」タイトルとユーザー情報の間に無駄な空白がない反面、セクション間の区切りが不明瞭

### 1-6. NotFound（390px）

**良い点**:
- シンプルで分かりやすいレイアウト
- 2 つの CTA ボタン（ダッシュボード / ログイン）が明確

**問題点**:
- `text-4xl` の見出しが 375px で改行が入り、バランスが悪い
- ボタンの高さ（`py-2` ≒ 32px）が 44px タップターゲットに未達
- 上部余白（`py-16`）がモバイルでは過剰

# Part 2: 要件

## 2. ビジョン

**「どのページでも、スマホで迷わず・ストレスなく操作できる」**

v3.0 は学習の核心（エディタ・テスト・チャレンジ）を最適化した。v3.1 はその周辺ページ（ダッシュボード・カリキュラム・プロフィール・認証・ヘッダー）を、同じ品質基準でモバイル対応する。

## 3. スコープ

### v3.1 対象（本書）

| ページ | 変更レベル | 概要 |
|--------|-----------|------|
| ダッシュボード | 中 | カードレイアウト再構成・情報密度調整・スクロール量削減 |
| カリキュラム | 小 | アコーディオン余白調整・タップターゲット拡大・階層強化 |
| プロフィール | 中 | スタッツ 2×2 化・バッジ折りたたみ・セクション整理 |

## 4. 機能要件

### 4-1. ダッシュボード

#### ウェルカムカード
- モバイル時は Coden ロゴを非表示にし、テキストのみでコンパクトに
- `py-8` → `py-5` でモバイル時の縦幅を削減

#### 学習コースセクション
- カテゴリカード（React, TypeScript）とフィーチャーカード（ベースヌック, デイリー）のスタイルを統一
- 全カードを同一のカードコンポーネントで表示し、アイコン + タイトル + 進捗の横並びレイアウトに
- モバイルでは 1 カラム、sm 以上で 2 カラム

#### スキルアップセクション
- 4 カード（コードドクター等）を 2×2 グリッドに変更（モバイル）
- アイコン + タイトルのみのコンパクト表示、説明文はモバイルで非表示

#### 学習ステータスセクション
- スタッツカード 2 列を維持
- プログレスバーの高さを `h-2` → `h-3` に拡大
- バッジ数表示をタップ可能にし、プロフィールページへ遷移

#### スクロール量削減
- 上記変更により、全体のスクロール量を現状の約 70% に削減することを目標とする

### 4-2. カリキュラム

#### アコーディオン
- モバイル時のパディングを `px-5 py-4` → `px-4 py-3` に調整
- アコーディオンヘッダーのタップターゲットを行全体に拡大

#### カテゴリ階層
- カテゴリヘッダー（React / TypeScript）に背景色または左ボーダーを追加し、視覚的階層を強化
- 練習モードセクションとの境界を `<hr>` または `border-t` で明確化

#### ステップリンク
- ステップリンクのタップターゲットを行全体に拡大（`block` + `min-h-[44px]`）
- ステップ番号とタイトルの間隔を `gap-3` に統一

### 4-3. プロフィール

#### スタッツカード
- モバイル時に 2×2 グリッド（`grid-cols-2`）に変更
- カードの高さを均等にし、数値を中央寄せ

#### 実績バッジ
- デフォルトで獲得済みバッジのみ表示し、「未獲得を表示」トグルで展開する
- 未獲得バッジは背景 `opacity-40` + グレースケールでより明確に区別
- バッジ一覧を 2 カラムグリッドに変更（モバイル）

#### セクション整理
- 「ダッシュボードへ戻る」リンクを左上に移動し、パンくず的な配置に
- ポイント履歴セクションを折りたたみ可能にする（デフォルトは閉じた状態）

### 4-4. ログイン / サインアップ

#### 余白調整
- `py-16` → `py-8 sm:py-16` でモバイル時の上部余白を削減
- `px-6` → `px-4 sm:px-6` でモバイル時の左右余白を調整

#### 入力フィールド
- `py-2` → `py-3` + `min-h-[44px]` でタップターゲットを確保
- `autocomplete` 属性を追加（`email`, `current-password`, `new-password`）

#### エラー表示
- エラーメッセージをフォーム上部に `role="alert"` で表示
- 赤ボーダー + 背景色で視認性を確保

#### ビジュアル
- Coden ロゴとタイトルを中央寄せに統一
- 「はじめて利用しますか？」テキストのコントラスト確認

### 4-5. AppHeader ドロワー

#### フルスクリーン化
- `w-72` → `w-full`（モバイル時はフルスクリーンドロワー）
- 背景オーバーレイは維持するが、タップで閉じる動作を確認

#### タップターゲット
- 全ナビゲーションリンクを `block min-h-[44px] flex items-center` に変更
- テキストだけでなく行全体をタップ可能に

#### セクション分離
- ユーザー情報、メインナビ、練習モード、ユーティリティ（プロフィール・ログアウト）をセクション `border-b` で明確に区切る
- 各セクションに小さなヘッダーラベル（`text-xs text-slate-400`）を追加

#### クローズ動作
- ドロワー外タップ（オーバーレイ）で閉じる
- スワイプ右で閉じる動作は v3.1 スコープ外（将来拡張）

### 4-6. NotFound

#### レスポンシブ見出し
- `text-4xl` → `text-2xl sm:text-4xl` で小画面での改行を回避

#### ボタン
- `py-2` → `py-3 min-h-[44px]` でタップターゲットを確保

#### 余白
- `py-16` → `py-8 sm:py-16` でモバイル時の上部余白を削減

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| タッチ操作 | 全インタラクティブ要素のタップターゲット 44×44px 以上（v3.0 基準の継続） |
| パフォーマンス | 追加の JS バンドルなし（CSS 調整が中心のため） |
| アクセシビリティ | v3.0 で確立した a11y パターン（role, aria-label 等）を維持 |
| テスト | 既存 695 テスト全 PASS + 新規テスト追加 |
| ブラウザ | iOS Safari 16+, Chrome Android 最新 2 バージョン |
| PC への影響 | PC レイアウトの変更は最小限（sm/md 以上のブレークポイントで既存スタイルを維持） |
